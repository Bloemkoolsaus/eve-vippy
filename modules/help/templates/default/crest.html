Authentication - EVE SSO (Single-Sign-On)

SSO and Vippy

You will now use your EVE login to login to Vippy.
This will give Vippy access to CREST. Under no circumstances is Vippy able to read or otherwise get your EVE login information.
See this dev blog for more details on how the sso works and why it is secure.

CREST and Vippy

With the death of the Ingame Browser, Vippy relies on CREST for authentication and tracking. With CREST, Vippy can confirm that you are who you say you are and can track your location for valuable intel on your map.
Authentication was previously done with API keys. Although they still work, and will work for quite some time, I opted to replace them for CREST, since we need CREST anyway. No more juggling of API keys :)
Tracking was previously done with headers from the ingame browser. Now that the IGB is going away, we need to use CREST to get your location and active ship-type.

CREST Tracking

CREST Fleet API

Vippy's main method of tracking is via the CREST fleet api. This requires you to add your fleet to Vippy.
Fleet api has a couple of limits:
  • When your fleet changes boss, the new boss will have to re-add the fleet!!
  • Fleet information can only updated every 30 seconds (this is the same lag as ingame)
    Read character-location-api section to learn how to refresh your location faster.
Adding your fleet:
At the bottom of the map there is a button 'Add Fleet', use that to add your fleet. You will need to enter the fleet boss and the CREST link for your fleet. You can find your link via the fleet window ingame.

CREST Character Location API

With those restrictions on the fleet api, why are we even using that?
Ship type
The character location api does not provide any information about your current shiptype. We cannot use this to log mass on wormholes for the jumplog.
Rate limits
Authenticated CREST (wich is what we use here) has a really, really (like really) low rate limit. This means Vippy can only do so many request in a given time. If Vippy needs to use the character location api to track all active characters, and honor the rate limit imposed by CCP, it would need 15 minutes for a round trip.
Add to that, we need to also call this api to check wether or not a character is even logged in!

Solution!

To get around the rate limit and the slow update rate of the fleet api there are now 2 new buttons on the map page. These buttons will update your location NOW, using the character-location api. This is usefull when you've just jumped into a new hole and want to start scanning right away.
Click the button, choose a toon and Vippy will try to use the character-location api to fetch your current location. Ofcourse, your toon needs to have a valid CREST token. Check your character status here
In your account settings you can select your dedicated scan-alt. If selected, you will have a second button that automatically selects your scan alt and refreshed it's location.
To prevent abuse, these buttons will be rate limited!!
